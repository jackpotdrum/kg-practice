# SQL模擬試験 #1（Medium+）

- 実施日: 2026-04-15
- テーマ: 連続ログイン日数（3日連続 streak 判定）
- 使用DB: PostgreSQL

## 問題文
あなたは、あるWebサービスのログイン履歴データを分析する担当です。  
サービス運営チームは「ユーザーの継続利用」を評価するため、3日連続でログインしたユーザーを抽出したいと考えています。

ただし、実データには次の特徴があります。

- 同じユーザーが同じ日に複数回ログインすることがある
- 端末が異なっていても、同日であれば「1ログイン日」として扱う
- 連続判定は「日付が1日ずつ連続していること」とする

タスクは、3日連続ログインを満たすすべての区間を出力するSQLを書くこと。  
1ユーザーが複数の3日連続区間を持つ場合は、すべて出力する。

## テーブル定義

### Logins
- `user_id` (INT): ユーザーID
- `login_date` (DATE): ログイン日
- `device_id` (VARCHAR): ログイン端末ID

## 出力要件 / 並び順
### 出力列
- `user_id`
- `streak_start`（3日連続区間の開始日）
- `streak_end`（3日連続区間の3日目）

### 並び順
1. `user_id` 昇順
2. `streak_start` 昇順

## あなたの解答
```sql
select user_id, distinct login_date as streak_start, login_date + interval '2 day' as streak_end fron Logins where streak_start + interval '1 day' in login_date and streak_start + interval '2 day' in login_date;
```

## 採点結果
- 総合: 35 / 100

### 内訳
- 正しさ: 25 / 60
- 可読性: 5 / 20
- 実行効率: 5 / 20

## 添削ポイント
1. `SELECT user_id, distinct ...` は構文エラー。`DISTINCT` は `SELECT DISTINCT ...` の形で使う。  
2. `fron` は `from` のタイプミス。  
3. `WHERE` 句で `streak_start`（同一SELECT内の別名）を参照しているが不可。  
4. `IN` の使い方が不正（`x IN (値リスト or サブクエリ)` が正しい）。  
5. 同日複数ログインを1日として扱う前処理（重複排除）が必要。

## 模範解答
```sql
WITH dedup AS (
  SELECT DISTINCT user_id, login_date
  FROM Logins
)
SELECT
  d.user_id,
  d.login_date AS streak_start,
  d.login_date + INTERVAL '2 day' AS streak_end
FROM dedup d
WHERE EXISTS (
  SELECT 1
  FROM dedup d2
  WHERE d2.user_id = d.user_id
    AND d2.login_date = d.login_date + INTERVAL '1 day'
)
AND EXISTS (
  SELECT 1
  FROM dedup d3
  WHERE d3.user_id = d.user_id
    AND d3.login_date = d.login_date + INTERVAL '2 day'
)
ORDER BY d.user_id, streak_start;
```

## 別解
```sql
WITH dedup AS (
  SELECT DISTINCT user_id, login_date
  FROM Logins
)
SELECT
  d1.user_id,
  d1.login_date AS streak_start,
  d3.login_date AS streak_end
FROM dedup d1
JOIN dedup d2
  ON d2.user_id = d1.user_id
 AND d2.login_date = d1.login_date + INTERVAL '1 day'
JOIN dedup d3
  ON d3.user_id = d1.user_id
 AND d3.login_date = d1.login_date + INTERVAL '2 day'
ORDER BY d1.user_id, streak_start;
```

## 間違えた原因（自己分析）
- 構文（DISTINCT位置、IN句）と論理（存在確認）の両方が混ざって崩れた。  
- まず「重複排除 → 連続存在確認」の2段階に分けるべきだった。
