# SQL模擬試験 #13（Hard）

- 実施日: 2026-04-29
- テーマ: Cohort retention（D1/D3/D7）+ 日付正規化 + 条件付きユニークカウント
- 使用DB: PostgreSQL

## 問題文
2026-03-01〜2026-03-07に登録したユーザーをコホートとして、登録日ごとに Day1 / Day3 / Day7 リテンションを集計する問題。

要件:
- Day1: 登録日の翌日にログイン
- Day3: 登録日の3日後にログイン
- Day7: 登録日の7日後にログイン
- 同一ユーザーの同日複数ログインは1回扱い
- 出力: signup_date, cohort_size, D1/D3/D7 の人数と率
- 並び順: signup_date 昇順

## テーブル定義
### users
- user_id (BIGINT): ユーザーID
- signup_at (TIMESTAMP): 登録日時

### logins
- login_id (BIGINT): ログインイベントID
- user_id (BIGINT): ユーザーID
- login_at (TIMESTAMP): ログイン日時

## 出力要件 / 並び順
### 出力列
- signup_date
- cohort_size
- d1_retained_users
- d1_retention_rate
- d3_retained_users
- d3_retention_rate
- d7_retained_users
- d7_retention_rate

### 並び順
1. signup_date ASC

## あなたの解答
```sql
with dedup as (
    select distinct user_id, login_at
    from logins
)
select
    u.signup_at as signup_date,
    sum(u.signup_at) as cohort_size,
    sum(d.login_at + interval '1 day') as d1_retained_users,
    round(d1_retained_users / cohort_size, 4) as d1_retention_rate,
    sum(d.login_at + interval '3 day') as d3_retained_users,
    round(d3_retained_users / cohort_size, 4) as d3_retention_rate,
    sum(d.login_at + interval '7 day') as d7_retained_users,
    round(d7_retained_users / cohort_size, 4) as d7_retention_rate
from dedup as d
join users as u on d.user_id = u.user_id
where signup_date >= timestamp '2026-03-01 00:00:00'
    and signup_date < timestamp '2026-03-08 00:00:00'
group by signup_date
order by signup_date asc
```

## 添削ポイント
1. dedup は login_at のままではなく login_at::date で日単位重複排除が必要。  
2. 人数を出す列に SUM を使ってしまっている。人数は COUNT 系で集計する。  
3. 同一SELECT内の別名（signup_date, d1_retained_users など）を WHERE や式中で直接使えない。CTEで段階化する。  
4. signup_at は timestamp のため、signup_at::date に正規化して日付粒度で group by する。  
5. DayN は「ちょうどN日後」判定で実装する。  
6. コホート母数は users 起点で保持するため LEFT JOIN を使う方が安全。

### LEFT JOINを使う理由（今回の問題）
- コホート分析では、対象期間に登録した全ユーザーを分母（cohort_size）として残す必要がある。
- `JOIN`（INNER JOIN）だとログインが1回もないユーザーが結合結果から落ち、分母が過小になる。
- `LEFT JOIN` なら `users` 側を全件保持し、ログインがないユーザーは右側が `NULL` で残る。
- このため D1/D3/D7 の人数と率を正しく計算できる。

## 模範解答
```sql
with cohort as (
    select
        u.user_id,
        u.signup_at::date as signup_date
    from users u
    where u.signup_at >= timestamp '2026-03-01 00:00:00'
      and u.signup_at <  timestamp '2026-03-08 00:00:00'
),
login_days as (
    select distinct
        l.user_id,
        l.login_at::date as login_date
    from logins l
),
joined as (
    select
        c.signup_date,
        c.user_id,
        ld.login_date
    from cohort c
    left join login_days ld
      on ld.user_id = c.user_id
),
agg as (
    select
        signup_date,
        count(distinct user_id) as cohort_size,
        count(distinct case when login_date = signup_date + 1 then user_id end) as d1_retained_users,
        count(distinct case when login_date = signup_date + 3 then user_id end) as d3_retained_users,
        count(distinct case when login_date = signup_date + 7 then user_id end) as d7_retained_users
    from joined
    group by signup_date
)
select
    signup_date,
    cohort_size,
    d1_retained_users,
    round(d1_retained_users::numeric / cohort_size, 4) as d1_retention_rate,
    d3_retained_users,
    round(d3_retained_users::numeric / cohort_size, 4) as d3_retention_rate,
    d7_retained_users,
    round(d7_retained_users::numeric / cohort_size, 4) as d7_retention_rate
from agg
order by signup_date asc;
```

## 別解
```sql
with cohort as (
  select user_id, signup_at::date as signup_date
  from users
  where signup_at >= timestamp '2026-03-01 00:00:00'
    and signup_at <  timestamp '2026-03-08 00:00:00'
),
login_days as (
  select distinct user_id, login_at::date as login_date
  from logins
)
select
  c.signup_date,
  count(distinct c.user_id) as cohort_size,
  count(distinct case when exists (
    select 1
    from login_days ld
    where ld.user_id = c.user_id
      and ld.login_date = c.signup_date + 1
  ) then c.user_id end) as d1_retained_users,
  round(
    count(distinct case when exists (
      select 1 from login_days ld
      where ld.user_id = c.user_id
        and ld.login_date = c.signup_date + 1
    ) then c.user_id end)::numeric
    / count(distinct c.user_id), 4
  ) as d1_retention_rate,
  count(distinct case when exists (
    select 1 from login_days ld
    where ld.user_id = c.user_id
      and ld.login_date = c.signup_date + 3
  ) then c.user_id end) as d3_retained_users,
  round(
    count(distinct case when exists (
      select 1 from login_days ld
      where ld.user_id = c.user_id
        and ld.login_date = c.signup_date + 3
    ) then c.user_id end)::numeric
    / count(distinct c.user_id), 4
  ) as d3_retention_rate,
  count(distinct case when exists (
    select 1 from login_days ld
    where ld.user_id = c.user_id
      and ld.login_date = c.signup_date + 7
  ) then c.user_id end) as d7_retained_users,
  round(
    count(distinct case when exists (
      select 1 from login_days ld
      where ld.user_id = c.user_id
        and ld.login_date = c.signup_date + 7
    ) then c.user_id end)::numeric
    / count(distinct c.user_id), 4
  ) as d7_retention_rate
from cohort c
group by c.signup_date
order by c.signup_date asc;
```

## 間違えた原因（自己分析）
- SUM と COUNT の役割を混同し、人数算出ロジックが崩れた。
- date / timestamp の型差を意識せず式を書いた。
- 同一SELECT内別名の参照制約を把握できていなかった。
