# SQL模擬試験 1（Medium-Hard）
- 実施日: 2026-04-30
- テーマ: カテゴリ別Top-Nランキング（優良購入者抽出）/ 集約 + Window関数
- 使用DB: PostgreSQL

## 問題文
あなたはECサービスの分析担当です。2026-04-01 から 2026-04-30 の期間における「カテゴリ別の優良購入者ランキング」を作成してください。
優良購入者の定義は「そのカテゴリで、完了済み注文を2件以上持つユーザー」です。
ランキングはカテゴリごとに作成し、各カテゴリの上位2名のみを出力してください。

## テーブル定義
### orders
- order_id: bigint, 注文ID（主キー）
- user_id: bigint, ユーザーID
- ordered_at: timestamp, 注文日時
- status: text, 注文状態（completed, cancelled など）

### order_items
- order_id: bigint, 注文ID（orders.order_id への外部キー）
- product_id: bigint, 商品ID
- category_id: text, カテゴリID
- quantity: integer, 購入数量
- unit_price: integer, 単価（税抜）

## 出力要件 / 並び順
### 出力列
- category_id
- rank_in_category
- user_id
- order_count
- total_amount
- latest_order_at

### 並び順
- category_id 昇順
- rank_in_category 昇順

### 注意事項
- 対象期間は 2026-04-01 00:00:00 以上 2026-05-01 00:00:00 未満
- status = completed の注文のみ対象
- order_count はカテゴリ内での distinct order_id
- rankの優先順位: total_amount DESC, latest_order_at DESC, user_id ASC
- 各カテゴリ rank_in_category <= 2 のみ出力

## あなたの解答
```sql
with period_orders as (
    select 
        o.order_id,
        o.user_id,
        o.ordered_at,
        o.status
    from orders as o
    where o.ordered_at >= timestamp '2026-04-01 00:00:00'
        and o.ordered_at < timestamp '2026-05-01 00:00:00'
        and o.status = 'completed'
)
latest_orders as (
    select
        oi.category_id as category_id,
        po.user_id as user_id,
        po.ordered_at as ordered_at
    from period_orders as po
    join order_items as oi on po.order_id = oi.order_id
    group by category_id, user_id
    order by ordered_at desc
    limit 1
)
agg as (
    select
        oi.category_id as category_id,
        po.user_id as user_id,
        count(distinct po.order_id) as order_count,
        sum(oi.quantity * oi.unit_price) as total_amount
    from period_orders as po
    join order_items as oi on po.order_id = oi.order_id
    group by category_id, user_id
)
agg2 as (
    select
        agg.category_id as category_id,
        agg.user_id as user_id,
        agg.order_count as order_count,
        agg.total_amount as total_amount
        lo.ordered_at as latest_order_at
    from agg
    join latest_orders as lo
        on agg.category_id = lo.category_id
        and agg.user_id = lo.user_id
)
ranked as (
    select
        category_id,
        user_id,
        order_count,
        total_amount,
        latest_order_at
        row_number() over (
            partition by category_id
            order by total_amount desc, latest_order_at desc, user_id asc
        ) as rank_in_category
    from agg2
)
select
    category_id,
    rank_in_category,
    user_id,
    order_count,
    total_amount,
    latest_order_at
from ranked
where rank_in_category <= 2
order by category_id asc, rank_in_category asc, user_id asc;
```

## 添削ポイント
- 構文面（いったん除外しても）以外の論点:
  - latest_orders の設計が「カテゴリ×ユーザーごとの最新日時」を安定して返せない形になっていた
  - 優良購入者条件（order_count >= 2）が未適用
- 良かった点:
  - 期間 + status の先絞り込み
  - order_count を count(distinct order_id) で定義
  - row_number の順位条件の方向性が正しい
- 改善の核:
  - 同一粒度（category_id, user_id）で取れる指標（order_count, total_amount, latest_order_at）は1つの集約CTEに寄せる

## 別解
```sql
with period_orders as (
    select o.order_id, o.user_id, o.ordered_at
    from orders o
    where o.ordered_at >= timestamp '2026-04-01 00:00:00'
      and o.ordered_at <  timestamp '2026-05-01 00:00:00'
      and o.status = 'completed'
),
base as (
    select
        oi.category_id,
        po.user_id,
        po.order_id,
        po.ordered_at,
        sum(oi.quantity * oi.unit_price) as order_amount
    from period_orders po
    join order_items oi on oi.order_id = po.order_id
    group by oi.category_id, po.user_id, po.order_id, po.ordered_at
),
agg as (
    select
        category_id,
        user_id,
        count(*) as order_count,
        sum(order_amount) as total_amount,
        max(ordered_at) as latest_order_at
    from base
    group by category_id, user_id
),
ranked as (
    select
        *,
        row_number() over (
            partition by category_id
            order by total_amount desc, latest_order_at desc, user_id asc
        ) as rank_in_category
    from agg
    where order_count >= 2
)
select
    category_id,
    rank_in_category,
    user_id,
    order_count,
    total_amount,
    latest_order_at
from ranked
where rank_in_category <= 2
order by category_id, rank_in_category;
```

## 学びの要点（再発防止チェックリスト）
- 集約の粒度を先に言語化する（このCTEの1行は何か）
- 最新日時が欲しいだけなら max(timestamp) をまず検討する
- 「最新行の他カラムも必要」なときだけ row_number で行特定に切り替える
- Top-N問題では rank条件と最終sort条件を分けて確認する
- 件数条件（今回なら order_count >= 2）を入れ忘れない

## 次回の課題（1〜2問）

2. セッション化（30分ギャップ）+ ユーザー別Top-N
