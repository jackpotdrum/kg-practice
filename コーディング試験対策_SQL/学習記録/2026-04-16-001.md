# SQL模擬試験 #2（Medium）

- 実施日: 2026-04-16
- テーマ: 期間抽出 + 顧客別集計 + HAVING判定
- 使用DB: PostgreSQL（MySQLでも同等に実装可）

## 問題文
ECサイトの売上テーブル `sales` を使って、2026年3月の顧客別売上を分析してください。
運用チームからは「高額購入顧客を抽出したい」という依頼が来ています。

要件は次の通りです。
- 集計対象は2026年3月の注文のみ
- 顧客ごとに売上金額を合計する
- 合計金額が50000以上の顧客だけを抽出する
- 合計金額の高い順で表示する

## テーブル定義
### sales
- `id` (INT): 注文ID
- `customer_id` (INT): 顧客ID
- `order_date` (DATE): 注文日
- `amount` (NUMERIC): 購入金額

## 出力要件 / 並び順
### 出力列
- `customer_id`
- `total_amount`

### 並び順
1. `total_amount` 降順

## あなたの解答
```sql
select customer_id, sum(amount) as total_amount from sales
where (select sum(amount) from sales) > 50000
and order_date >= '2026-03-01' and order_date < '2026-04-01'
group by costomer_id
order by total_amount desc;
```

## 採点結果
- 総合: 62 / 100

### 内訳
- 正しさ: 34 / 60
- 可読性: 14 / 20
- 実行効率: 14 / 20

## 添削ポイント
1. 50000以上の条件は集計後判定なので `WHERE` ではなく `HAVING` を使う。  
2. `(select sum(amount) from sales) > 50000` は「全体合計」の判定であり、顧客ごとの判定になっていない。  
3. `group by costomer_id` はスペルミス（正: `customer_id`）。

## 模範解答
```sql
SELECT
  customer_id,
  SUM(amount) AS total_amount
FROM sales
WHERE order_date >= '2026-03-01'
  AND order_date < '2026-04-01'
GROUP BY customer_id
HAVING SUM(amount) >= 50000
ORDER BY total_amount DESC;
```

## 別解
```sql
WITH monthly AS (
  SELECT
    customer_id,
    SUM(amount) AS total_amount
  FROM sales
  WHERE order_date >= '2026-03-01'
    AND order_date < '2026-04-01'
  GROUP BY customer_id
)
SELECT
  customer_id,
  total_amount
FROM monthly
WHERE total_amount >= 50000
ORDER BY total_amount DESC;
```

- [ ] 行を絞る条件は `WHERE`、集計結果を絞る条件は `HAVING` に置いたか  
- [ ] 判定単位（全体 or 顧客ごと）をSQLで一致させたか  
- [ ] `GROUP BY` 列名のスペルを最終チェックしたか

## 間違えた原因（自己分析）
- 集計関数の条件を `WHERE` に書いてしまい、処理順の理解が崩れた。  
- サブクエリで「全体合計」を見てしまい、問題の単位（顧客ごと）とずれた。
