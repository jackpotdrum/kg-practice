# SQL模擬試験 #12（Hard）

- 実施日: 2026-04-16
- テーマ: Top-N per group + 最新重複排除 + Window関数による順位付け
- 使用DB: PostgreSQL（MySQL 8.0+でも概ね実装可）

## 問題文
ECプラットフォームの2026年3月販売実績から、カテゴリごとの売上上位商品を抽出する問題。

要件:
- 対象期間は 2026-03-01 00:00:00 以上 2026-04-01 00:00:00 未満
- 対象注文は `orders.status = 'paid'` のみ
- `products.deleted_at is null` の商品のみ
- `order_items` は同一 `(order_id, product_id)` の重複があり、`ingested_at` 最新1件のみ有効
- カテゴリごとに上位2商品を返す
- 順位は `total_sales desc` -> `total_qty desc` -> `product_id asc` で決定

## テーブル定義
### orders
- `order_id` (INT): 注文ID
- `user_id` (INT): ユーザーID
- `ordered_at` (TIMESTAMP): 注文日時
- `status` (TEXT): 注文ステータス（paid, cancelled, refunded など）

### order_items
- `item_event_id` (INT): 明細イベントID（一意）
- `order_id` (INT): 注文ID
- `product_id` (INT): 商品ID
- `quantity` (INT): 数量
- `unit_price` (NUMERIC): 注文時単価
- `ingested_at` (TIMESTAMP): DWH取り込み時刻

### products
- `product_id` (INT): 商品ID
- `category` (TEXT): カテゴリ名
- `product_name` (TEXT): 商品名
- `deleted_at` (TIMESTAMP NULL): 削除日時（NULLなら有効）

## 出力要件 / 並び順
### 出力列
- `category`
- `rank_in_category`
- `product_id`
- `product_name`
- `total_qty`
- `total_sales`

### 並び順
1. `category` 昇順
2. `rank_in_category` 昇順
3. `product_id` 昇順

## あなたの解答
```sql
select
    p.category,
    row_number() over (order by sum(oi.quantity * oi.unit_price desc)) as rank_in_category,
    p.product_id,
    p.product_name,
    count(*) as total_qty,
    sum(oi.quantity * oi.unit_price) as total_sales
from order_items as oi
join orders as o in oi.order_id = o.order_id
join products as p in oi.product_id = p.product_id
where oi.ordered_at >= '2026-03-01' and oi.ordered_at < '2026-04-01' and p.deleted_at is null and o.status = 'paid'
group by p.category
order by total_sales desc, total_qty desc, p.product_id asc
limit 2;
```

## 採点結果
- 総合: 33 / 100

### 内訳
- 正しさ: 20 / 70
- 可読性: 8 / 15
- 実行効率: 5 / 15

## 添削ポイント
1. `JOIN ... IN ...` は構文誤り。正しくは `JOIN ... ON ...`。
2. 日時列は `order_items.ordered_at` ではなく `orders.ordered_at` を使う。
3. 重複排除（同一 `(order_id, product_id)` の最新1件）を先に実装する必要がある。
4. `row_number()` はカテゴリ内順位にするため `PARTITION BY category` が必要。
5. `total_qty` は `count(*)` ではなく `sum(quantity)` が要件。
6. `group by` 粒度が不足（商品単位で集計する必要あり）。
7. `limit 2` は全体上位2件になり、カテゴリごと上位2件にならない。

## 模範解答
```sql
with latest_item as (
    select
        oi.order_id,
        oi.product_id,
        oi.quantity,
        oi.unit_price,
        row_number() over (
            partition by oi.order_id, oi.product_id
            order by oi.ingested_at desc, oi.item_event_id desc
        ) as rn
    from order_items oi
),
base as (
    select
        p.category,
        p.product_id,
        p.product_name,
        li.quantity,
        li.unit_price
    from latest_item li
    join orders o
      on o.order_id = li.order_id
    join products p
      on p.product_id = li.product_id
    where li.rn = 1
      and o.status = 'paid'
      and o.ordered_at >= timestamp '2026-03-01 00:00:00'
      and o.ordered_at <  timestamp '2026-04-01 00:00:00'
      and p.deleted_at is null
),
agg as (
    select
        category,
        product_id,
        product_name,
        sum(quantity) as total_qty,
        sum(quantity * unit_price) as total_sales
    from base
    group by category, product_id, product_name
),
ranked as (
    select
        category,
        product_id,
        product_name,
        total_qty,
        total_sales,
        row_number() over (
            partition by category
            order by total_sales desc, total_qty desc, product_id asc
        ) as rank_in_category
    from agg
)
select
    category,
    rank_in_category,
    product_id,
    product_name,
    total_qty,
    total_sales
from ranked
where rank_in_category <= 2
order by category asc, rank_in_category asc, product_id asc;
```

## 別解
```sql
with dedup as (
    select
        oi.order_id,
        oi.product_id,
        oi.quantity,
        oi.unit_price,
        row_number() over (
            partition by oi.order_id, oi.product_id
            order by oi.ingested_at desc
        ) as rn
    from order_items oi
),
agg as (
    select
        p.category,
        p.product_id,
        p.product_name,
        sum(d.quantity) as total_qty,
        sum(d.quantity * d.unit_price) as total_sales
    from dedup d
    join orders o on d.order_id = o.order_id
    join products p on d.product_id = p.product_id
    where d.rn = 1
      and o.status = 'paid'
      and o.ordered_at >= '2026-03-01'
      and o.ordered_at < '2026-04-01'
      and p.deleted_at is null
    group by p.category, p.product_id, p.product_name
)
select *
from (
    select
        a.*,
        row_number() over (
            partition by a.category
            order by a.total_sales desc, a.total_qty desc, a.product_id asc
        ) as rank_in_category
    from agg a
) t
where rank_in_category <= 2
order by category asc, rank_in_category asc, product_id asc;
```

## 間違えた原因（自己分析）
- いきなり完成SQLを書こうとして、粒度設計と処理順を飛ばしてしまった。
- GROUP BY と PARTITION BY の役割分担が曖昧なまま書き始めた。
